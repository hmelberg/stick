stick.

A tiny stick-figure animation language — describe a scene in JSON, watch it play as an animated SVG cartoon.

stick turns a single JSON document into a hand-drawn, animated stick-figure cartoon that plays in any browser — no build step, no server, no dependencies. It was designed so that a person or an AI can write the JSON: the language is small, semantic (move, say, mood), and the engine is deliberately forgiving — a bad event is skipped with a warning rather than blanking the screen. The result is fast, expressive storytelling: lectures, sketches, explainers, and gags, all from a few readable lines of JSON.

How it works.   JSON → compile() expands each timeline command into animation channels (keyframed value tracks) → the player samples every channel at the current time and draws an SVG frame. Three visual styles share one skeleton, so every command works in every style. Timing is relative by default (each event follows the previous one), with GSAP-style offsets for overlap and simultaneity.

1Figures & characters

Every figure is an articulated skeleton rendered in one of three switchable styles: sketch (hand-drawn wobbly "line boil", the default), ink (clean minimal stick), and toon (filled cartoon body with a coloured torso, neck and shoes). The same JSON renders in any style.

Identity

Archetypes (man, woman, kid, person) and stronger characters (professor, student, dancer). Per-figure colour, height, body colour.

Looks

Optional hair (short, tuft, long-with-curl, bun, sides), beard (full / goatee / stubble), glasses, and a fedora. Bald by default.

Full 360° facing

front · left · right · back or a continuous number; the turn command tweens smoothly through every in-between angle — face the class, then turn to the board.

Bust mode

body:"bust" renders a floating head + neck + arms (no torso/legs) — a talking-head explainer that still gestures.

Moods & expression

Nine mood presets (happy, bored, thinking, angry, sad, surprised, sleepy…) that set face and posture and idle motion, plus fine expression channels (smile, brows, gaze, eye-open, mouth).

Living idle

Figures breathe, sway and blink on their own, tuned by mood, so even a still character feels alive. Faces get small per-figure variation.

2Motion & gesture

  • Locomotion — move with walk / run / slide / moonwalk; duration auto-derived from distance, legs animate automatically, the figure auto-faces travel. The walk cycle has a real gait: knee bend and foot-lift on the swing leg, a toe-point through the swing, and a heel-strike at landing.
  • Age-based movement — age: kid / adult / elderly re-times the same gait (kids bounce with high knees, the elderly shuffle with a stoop) without changing size.
  • Gestures & clips — a dozen built-in clips (wave, nod, bow, shrug, facepalm, victory, thinkChin, coolLean…) plus your own reusable clips.
  • Pose & IK — semantic raiseArm/liftLeg, point, lookAt, reachTo (hand reaches and holds a point), pin (lock feet/hands), hop, mitten hand shapes, and low-level joints FK for special choreography.

3Speech & sound

say shows a speech bubble and animates the mouth; scene.caption is narrator text; emote floats an inner-state symbol (? ! ♥ ♪ zzz idea…). When the viewer turns sound on, lines are spoken aloud via the browser's speech engine — default British-English male, with per-figure voice overrides for gender, language/accent (friendly names like french or BCP-47 codes), pitch, rate, volume, and a sing-song mode. In a crowd, each figure is automatically given a distinct voice so they don't all sound alike.

4Camera & cinematography

Frame anything

camera.focus auto-fits a target so it fills the frame: a whole figure, a board, a face close-up ("ana.face") for subtle emotion, or an arbitrary region.

Move & cut

panTo, zoom, instant cut, reset, plus follow to track a walking figure for side-scrolling shots.

Mood & impact

Dutch-angle tilt and camera shake for tension and impact beats.

Depth & settings

Optional parallax (far layers move slower), and scene.fade / scene.cut to change the backdrop to a whole new setting.

5Held props

A figure can hold a prop in a hand and it follows the hand every frame. There's a built-in library (apple, ball, cup, coffee, book, phone, balloon, briefcase, sign, and directional gun / sword / wand / pencil that aim along the arm), and the LLM can also define a custom prop inline — any shape, with a grip point and an optional "aim along the forearm" flag. Five verbs: give, drop, throw (a real parabolic arc with spin), pickUp, and handOff between figures.

6Animatable objects

For non-figure things that move, grow, spin, fade, or recolour — balls, boxes, suns, signs — declare objects (any SVG shape) and animate them with a few verbs: appear/disappear, moveTo, arc (toss with spin), grow/shrink/scale, rotate/spin, color. Any object id is also a live point reference, so a figure can lookAt a moving ball — which makes juggling, bouncing and chases easy.

7Boards — teaching & presenting

A board is a chalk- or marker-styled panel you write markdown to, revealed with a left-to-right handwriting wipe (with a chalk hand that tracks the writing point), auto-scrolling when it overflows. It draws diagrams stroke-by-stroke (a built-in supply-&-demand chart, or custom axes/curves/lines/dots/labels), highlights elements or words while a character talks about them, and typesets real math — LaTeX ($…$, $$…$$) and AsciiMath — via MathJax. Coloured words, headings, bullets, and dividers are supported inline.

8Scenes & worlds

Built-in themes (classroom, street, bedroom, blank) or fully custom scenery from primitive shapes on back / mid / front layers, with named anchors you can reference anywhere a point is expected. Coordinates aren't limited to the 0–100 frame: place scenery far to the right, tile an "endless" background with repeat, and use camera.follow for a side-scrolling journey.

9Timeline & timing language

Events run sequentially by default — each starts when the previous ends — so most scenes need no explicit times at all. When you want overlap or simultaneity, a small GSAP-flavoured grammar covers it: "at": 2.5 (absolute), "+quick" / "-quick" (after/overlap the previous end), "<" (together with the previous), "<+fast" (just after it starts). Durations have friendly names (veryFast … verySlow) or raw seconds, and a target can be an array to drive several figures at once.

10Studio & export

Playground

Pick from ~20 bundled examples, edit the JSON live (Ctrl+Enter to render), scrub the timeline, Space to play/pause. Warnings list bad events without ever blanking the screen.

Import / export

Upload & download the JSON, or record the scene to a WebM video in real time — optionally capturing the spoken audio too.

AI authoring

Signed-in users get a Create panel: describe a scene in plain words and a Netlify edge function asks Claude to write the JSON, using the same reference this engine ships with.

Saved work

Created and uploaded animations are saved in the browser (localStorage) under "my animations".

★Feature ideas — now all shipped

These were proposed as easy wins; every one has since been built and is live in the engine. They fit the channel-based, forgiving architecture without re-plumbing it. New commands at a glance: scene.tint/scene.untint, ground shadows (scene.shadows), think, arrow/connect, move … via (path-following), burst (confetti), give … hand:"both" (two-handed) + inline custom & group props, sfx, and a .srt subtitle sidecar on WebM export — plus the earlier emotions, mood intensity, and per-figure variation. easy/moderate below mark how each turned out.

Ground shadows easy

A soft ellipse under each figure/object, scaled by height and "lift", instantly grounds everything and reads as light. Pure render-layer addition; no DSL change.

More emotions + intensity easy

Add crying (with animated tears), confused, worried, and an intensity modifier — mood: "angry", intensity: 1.5 or "very angry" — that scales the expression deltas. Extends the existing mood-preset table and expression channels.

Sound-effects library moderate

A tiny Web-Audio sfx command (boing, pop, whoosh, ding, thud) that pairs with hop, throw, camera.shake. Unlike speech it can be captured into the WebM export, so it strengthens recordings too.

Thought bubbles & connectors easy

A think command (cloud bubble vs. speech bubble) and simple arrows/connectors between two anchors — both are overlay variants of machinery that already exists, and both are high-value for explainers and diagrams.

Path-following motion easy

Let move / moveTo take a list of waypoints or a curve and sample it into keyframes — exactly how throw already builds its arc. Enables strolls, swoops, and looping patrols.

Scene tint / time-of-day easy

A tweenable full-frame colour overlay (scene.tint) for sunset, night, a red alert flash, or a dream wash. One rect + an opacity/colour channel.

Grouped / multi-shape props & objects moderate

Today a prop/object is a single shape. Allowing a group of shapes (one transform) removes the main authoring limit — richer props (a mug with steam, a flower with a stem) and composite objects.

Per-figure random variation easy

Use each figure's existing seed to nudge resting posture, gait phase, and idle timing, so a crowd never moves in lockstep. The voice system already does this for sound.

Confetti / particle burst moderate

A celebratory burst emote — a handful of small shapes that arc out and fade — reusing the object arc math. A natural "button" to end a happy scene.

Two-handed props & caption track export moderate

Hold one prop with both hands (a big sign, a steering wheel) by averaging the two hand points; and export an .srt/.vtt caption sidecar from the say lines alongside the WebM, for accessible, subtitled videos.
